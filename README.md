# 校园二手物品智能匹配平台

基于多特征融合推荐算法的校园二手物品交易与智能匹配平台。

## 项目简介

本平台是一个面向高校学生的二手物品交易与智能匹配平台。不同于传统商城系统，平台支持用户发布闲置商品、发布求购需求，并基于**多特征融合推荐算法**，从商品分类、关键词、价格、新旧程度、交易地点、用户行为、商品热度等多个维度，自动计算商品与求购需求的匹配度，直观展示匹配分数和匹配原因，提高校园二手交易的效率与体验。

```bash

```


### Token 使用方式

需要登录的接口在请求头中携带 Token：

```
Authorization: Bearer <token>
```

### 加密方式

- 密码使用 BCrypt 加密存储
- Token 使用 JWT（HS512），包含 userId、username、userType 信息
- Token 默认有效期 24 小时

### 图片上传目录说明

- 开发环境上传目录：`secondhand-backend/upload/`
- 生产环境上传目录：`/www/upload/`
- 按日期分目录存储，例如 `/upload/2026/06/xxx.jpg`
- 文件名使用 UUID，避免重名
- 支持 jpg、jpeg、png、webp 格式，限制 5MB

可在 `application.yml` 中配置：
```yaml
file:
  upload-path: ./upload/      # 上传目录
  access-prefix: /upload/     # 访问前缀
```

4. 筛选栏：关键词搜索、状态筛选（待审核/已通过/已拒绝/已关闭）、分类筛选
5. 列表为空时显示空状态提示

### 7. 求购审核通过和拒绝测试

1. pending 状态的求购显示「通过」和「拒绝」按钮
2. 点击「通过」→ 确认对话框 → 确认 → 状态变为「已通过」（绿色徽章）
3. 点击「拒绝」→ 填写拒绝原因 → 确认 → 状态变为「已拒绝」（红色徽章）
4. approved/rejected/closed 状态不显示审核按钮

### 8. 用户列表测试

1. 管理员登录后，点击左侧菜单「用户管理」
2. 进入 `/users` 页面，显示用户卡片列表
3. 每条卡片展示：ID、头像、用户名、昵称、手机号、邮箱、状态徽章（正常/已禁用）、注册时间
4. 顶部筛选栏：关键词搜索（用户名/昵称/手机号/邮箱）、状态筛选（正常/已禁用）
5. 点击「查询」刷新列表，「重置」清空筛选

### 9. 禁用用户测试

1. 在用户列表中找到状态为「正常」（绿色徽章）的用户
2. 右侧显示红色「禁用」按钮
3. 点击「禁用」→ 弹出确认框（含禁用提示说明）
4. 点击「确认禁用」→ 调用 `PUT /api/admin/user/status/{id}` `{ status: 0 }`
5. 提示"用户已禁用"，列表刷新，徽章变为红色「已禁用」

### 10. 启用用户测试

1. 在用户列表中找到状态为「已禁用」（红色徽章）的用户
2. 右侧显示绿色「启用」按钮
3. 点击「启用」→ 弹出确认框 → 点击「确认启用」
4. 调用 `PUT /api/admin/user/status/{id}` `{ status: 1 }`
5. 提示"用户已启用"，列表刷新，徽章变为绿色「正常」

### 11. 订单列表和状态筛选测试

1. 管理员登录后，点击左侧菜单「订单管理」
2. 进入 `/orders` 页面，显示订单卡片列表
3. 每条卡片展示：订单ID、状态徽章、商品标题/价格、买家/卖家信息、留言、创建/更新时间
4. 筛选栏：状态筛选（待处理/已接受/已拒绝/已完成/已取消）、买家ID、卖家ID、商品ID
5. 状态不同颜色徽章：待处理-黄、已接受-蓝、已拒绝-红、已完成-绿、已取消-灰
6. 订单只能查看，不提供管理员修改状态功能
