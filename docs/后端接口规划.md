### 智能匹配

#### 多特征融合评分规则（总分100）

| 维度 | 分值 | 说明 |
|------|------|------|
| 分类匹配 | 20分 | 商品与求购 categoryId 相同得满分 |
| 关键词匹配 | 30分 | 基于 Jaccard 相似度计算，提取 title/description 2-gram 和 keywords |
| 价格匹配 | 20分 | 商品价格在求购预算范围内得满分，超出按比例扣分 |
| 成色匹配 | 10分 | 新旧程度等级差越小分越高 |
| 地点匹配 | 10分 | 交易地点相同得满分，包含关系得6分 |
| 热度匹配 | 10分 | 基于浏览量和收藏量 |

#### 1. 根据求购匹配商品

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/match/goods-for-demand/{demandId}` | 根据求购匹配商品（需登录） |

**请求参数**：

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| limit | int | 10 | 返回条数 |

**权限**：只能对自己的求购进行匹配

**响应示例**：
```json
{
  "code": 200,
  "message": "success",
  "data": [
    {
      "goodsId": 5,
      "title": "二手自行车",
      "price": 200.00,
      "conditionLevel": "good",
      "mainImage": "/uploads/abc.jpg",
      "sellerNickname": "小刚",
      "matchScore": 85,
      "matchReasons": ["分类匹配", "关键词匹配(60%): 自行车,二手", "价格完全匹配(200元)", "成色接近"]
    }
  ]
}
```

### 智能推荐

#### 1. 个性化商品推荐

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/recommend/goods` | 个性化商品推荐（需登录） |

**请求参数**：

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| limit | int | 10 | 返回条数 |

**推荐策略**：
- 基于用户搜索历史、浏览历史、收藏记录、求购关键词构建用户画像
- 计算候选商品与用户画像的关键词 Jaccard 相似度（60分）
- 浏览/收藏过的商品额外加分（20分）
- 热度加成（10分）
- 无行为数据时返回热门商品（按 viewCount + favoriteCount × 3 排序）

#### 2. 相似商品推荐

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/recommend/similar/{goodsId}` | 相似商品推荐（匿名可访问） |

**请求参数**：

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| limit | int | 10 | 返回条数 |

**推荐策略**：基于分类（30分）、关键词（40分）、价格（15分）、成色（15分）计算相似度。

### 四、文件上传模块

#### 1. 上传图片

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/upload/image` | 上传图片（需登录，form-data） |

**请求参数**：`file`（multipart/form-data）

**响应示例**：
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "url": "/upload/2026/06/xxx.jpg"
  }
}
```

**限制**：
- 仅支持 jpg、jpeg、png、webp 格式
- 文件大小不超过 5MB
- 按日期分目录存储，文件名使用 UUID

**访问**：上传后可通过 `http://localhost:8080/upload/2026/06/xxx.jpg` 直接访问图片

| 权限级别 | 可访问接口 |

```json
{
  "code": 403,
  "message": "无权限",
```

## 统一响应格式

所有接口统一使用以下格式：

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

状态码：
- 200：成功
- 400：参数错误
- 401：未登录或 Token 已过期
- 403：无权限
- 404：资源不存在
- 500：服务器内部错误
