# 后端接口规划

## 当前已完成接口

### 健康检查

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/health` | 服务健康检查 |

**响应示例**：
```json
{
  "code": 200,
  "message": "success",
  "data": "secondhand-backend is running"
}
```

## 统一响应格式

所有接口统一使用以下格式：

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

## 认证方式

需要登录的接口在请求头携带 Token：

```
Authorization: Bearer <token>
```

## 分页请求参数

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| page | int | 1 | 当前页码 |
| size | int | 10 | 每页条数 |
| keyword | String | - | 搜索关键词 |
| categoryId | Long | - | 分类ID筛选 |
| sort | String | - | 排序方式 |
