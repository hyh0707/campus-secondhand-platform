# 数据库设计文档

- **表数量**：13 张

## 表详细设计

### 1. `user` 用户表

**作用**：存储平台所有注册用户的基本信息。

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGINT | 主键，自增 |
| `username` | VARCHAR(50) | 用户名，唯一索引，用于登录 |
| `password` | VARCHAR(255) | 密码（后续阶段改为BCrypt加密存储） |
| `nickname` | VARCHAR(50) | 昵称，可空 |
| `avatar` | VARCHAR(500) | 头像URL |
| `phone` | VARCHAR(20) | 手机号，唯一索引 |
| `email` | VARCHAR(100) | 邮箱，唯一索引 |
| `status` | TINYINT | 0-禁用，1-正常 |
| `create_time` | DATETIME | 创建时间 |
| `update_time` | DATETIME | 更新时间（自动更新） |

**关联关系**：
- `user.id` → `goods.user_id`：一个用户发布多个商品
- `user.id` → `demand.user_id`：一个用户发布多个求购需求
- `user.id` → `favorite.user_id`：一个用户收藏多个商品
- `user.id` → `trade_order.buyer_id` / `seller_id`：用户参与交易
- `user.id` → `browse_history.user_id`：用户浏览记录
- `user.id` → `search_history.user_id`：用户搜索记录
- `user.id` → `report.user_id`：用户举报记录

---

### 2. `admin` 管理员表

**作用**：存储后台管理系统管理员账号。

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGINT | 主键，自增 |
| `username` | VARCHAR(50) | 管理员用户名，唯一索引 |
| `password` | VARCHAR(255) | 密码（后续加密） |
| `nickname` | VARCHAR(50) | 管理员昵称 |
| `role` | VARCHAR(20) | admin-超级管理员，editor-内容编辑 |
| `status` | TINYINT | 0-禁用，1-正常 |
| `create_time` | DATETIME | 创建时间 |
| `update_time` | DATETIME | 更新时间 |

---

### 3. `category` 商品分类表

**作用**：存储商品分类，支持两级分类（parent_id 自关联）。

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGINT | 主键，自增 |
| `name` | VARCHAR(50) | 分类名称 |
| `parent_id` | BIGINT | 父分类ID，0表示顶级分类 |
| `icon` | VARCHAR(255) | 分类图标URL |
| `sort` | INT | 排序值，越小越靠前 |
| `status` | TINYINT | 0-禁用，1-启用 |
| `create_time` | DATETIME | 创建时间 |
| `update_time` | DATETIME | 更新时间 |

**关联关系**：
- `category.id` → `goods.category_id`：一个分类下有多个商品
- `category.id` → `demand.category_id`：一个分类下有多个求购需求
- `category.parent_id` → `category.id`：自关联，形成分类树

---

### 4. `goods` 商品表

**作用**：存储用户发布的闲置商品信息，是核心业务表。

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGINT | 主键，自增 |
| `user_id` | BIGINT | 发布用户ID |
| `category_id` | BIGINT | 所属分类ID |
| `title` | VARCHAR(200) | 商品标题 |
| `description` | TEXT | 商品描述 |
| `price` | DECIMAL(10,2) | 商品价格（元） |
| `condition_level` | VARCHAR(20) | 新旧程度（brand_new / like_new / good / moderate / worn） |
| `trade_location` | VARCHAR(200) | 交易地点 |
| `contact_info` | VARCHAR(200) | 联系方式 |
| `negotiable` | TINYINT | 是否可议价（0-否，1-是） |
| `view_count` | INT | 浏览次数（热度指标） |
| `favorite_count` | INT | 收藏次数（热度指标） |
| `status` | VARCHAR(20) | pending / approved / rejected / offline / sold |
| `create_time` | DATETIME | 创建时间 |
| `update_time` | DATETIME | 更新时间 |

**索引说明**：
- `idx_goods_user_id`：按用户查询发布的商品
- `idx_goods_category_id`：按分类筛选商品
- `idx_goods_price`：价格区间查询
- `idx_goods_condition`：按新旧程度筛选
- `FULLTEXT ft_goods_title_desc`：全文索引，用于关键词搜索匹配

**关联关系**：
- `goods.id` → `goods_image.goods_id`：一个商品有多张图片
- `goods.id` → `favorite.goods_id`：商品被多个用户收藏
- `goods.id` → `trade_order.goods_id`：商品产生交易意向
- `goods.id` → `browse_history.goods_id`：商品被浏览记录
- `goods.id` → `recommend_record.goods_id`：商品被推荐记录

---

### 5. `goods_image` 商品图片表

**作用**：存储商品的多张图片，第一张（sort=0）为主图。

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGINT | 主键，自增 |
| `goods_id` | BIGINT | 所属商品ID |
| `image_url` | VARCHAR(500) | 图片URL |
| `sort` | INT | 排序值，越小越靠前，sort=0 为主图 |
| `create_time` | DATETIME | 创建时间 |

---

### 6. `demand` 求购需求表

**作用**：存储用户发布的求购需求，是智能匹配的核心输入之一。

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGINT | 主键，自增 |
| `user_id` | BIGINT | 发布用户ID |
| `category_id` | BIGINT | 期望分类ID |
| `title` | VARCHAR(200) | 需求标题 |
| `description` | TEXT | 需求描述 |
| `min_price` | DECIMAL(10,2) | 最低接受价格 |
| `max_price` | DECIMAL(10,2) | 最高接受价格 |
| `expected_condition` | VARCHAR(20) | 期望新旧程度 |
| `expected_location` | VARCHAR(200) | 期望交易地点 |
| `keywords` | VARCHAR(500) | 关键词（逗号分隔），用于匹配计算 |
| `status` | VARCHAR(20) | pending / approved / rejected / closed |
| `create_time` | DATETIME | 创建时间 |
| `update_time` | DATETIME | 更新时间 |

**匹配算法涉及字段**：`category_id`（分类匹配）、`min_price` / `max_price`（价格匹配）、`expected_condition`（成色匹配）、`expected_location`（地点匹配）、`keywords`（关键词匹配）

---

### 7. `favorite` 收藏表

**作用**：记录用户收藏的商品，用于用户行为分析和个性化推荐。

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGINT | 主键，自增 |
| `user_id` | BIGINT | 用户ID |
| `goods_id` | BIGINT | 商品ID |
| `create_time` | DATETIME | 收藏时间 |

**唯一约束**：`(user_id, goods_id)` 防止重复收藏。

---

### 8. `trade_order` 交易意向表

**作用**：记录买家发起的交易意向，跟踪交易状态流转。

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGINT | 主键，自增 |
| `buyer_id` | BIGINT | 买家用户ID（发起交易意向） |
| `seller_id` | BIGINT | 卖家用户ID（商品发布者） |
| `goods_id` | BIGINT | 商品ID |
| `message` | VARCHAR(500) | 买家留言 |
| `status` | VARCHAR(20) | pending / accepted / rejected / completed / cancelled |
| `create_time` | DATETIME | 创建时间 |
| `update_time` | DATETIME | 更新时间 |

**状态流转**：
```
pending ──→ accepted ──→ completed
  │            │
  ├──→ rejected
  └──→ cancelled
```

---

### 9. `browse_history` 浏览记录表

**作用**：记录用户浏览商品的行为，用于用户画像分析和推荐算法。

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGINT | 主键，自增 |
| `user_id` | BIGINT | 用户ID |
| `goods_id` | BIGINT | 商品ID |
| `create_time` | DATETIME | 浏览时间 |

**用途**：分析用户兴趣偏好，计算商品热度（`goods.view_count` 可从此表聚合）。

---

### 10. `search_history` 搜索记录表

**作用**：记录用户搜索关键词，用于分析用户需求和关键词匹配。

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGINT | 主键，自增 |
| `user_id` | BIGINT | 用户ID |
| `keyword` | VARCHAR(200) | 搜索关键词 |
| `create_time` | DATETIME | 搜索时间 |

**用途**：提取用户兴趣关键词，用于推荐算法中的关键词维度匹配。

---

### 11. `recommend_record` 推荐记录表

**作用**：存储智能匹配算法的计算结果，是平台核心推荐功能的载体。

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGINT | 主键，自增 |
| `user_id` | BIGINT | 用户ID（求购需求发布者） |
| `demand_id` | BIGINT | 求购需求ID |
| `goods_id` | BIGINT | 匹配的商品ID |
| `match_score` | DECIMAL(5,2) | 匹配分数（0.00-100.00） |
| `match_level` | VARCHAR(20) | 匹配等级：high(>=70) / medium(40-69) / low(<40) |
| `match_reason` | VARCHAR(1000) | 匹配原因（JSON格式，包含各维度匹配详情） |
| `create_time` | DATETIME | 推荐时间 |

**匹配等级阈值**：
- **high (高匹配)**：match_score >= 70，推荐优先展示
- **medium (中匹配)**：40 <= match_score < 70，可供参考
- **low (低匹配)**：match_score < 40，通常不展示

---

### 12. `report` 举报表

**作用**：记录用户举报信息，用于平台内容管理和用户行为监督。

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGINT | 主键，自增 |
| `user_id` | BIGINT | 举报人用户ID |
| `target_type` | VARCHAR(20) | 举报目标类型：goods-商品，user-用户 |
| `target_id` | BIGINT | 举报目标ID |
| `reason` | VARCHAR(500) | 举报原因 |
| `status` | VARCHAR(20) | pending-待处理 / resolved-已处理 / dismissed-已驳回 |
| `create_time` | DATETIME | 举报时间 |
| `update_time` | DATETIME | 处理时间 |

---

### 13. `notice` 公告表

**作用**：存储平台公告，由管理员发布，展示给所有用户。

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGINT | 主键，自增 |
| `title` | VARCHAR(200) | 公告标题 |
| `content` | TEXT | 公告内容 |
| `status` | TINYINT | 0-下架，1-发布 |
| `create_time` | DATETIME | 创建时间 |
| `update_time` | DATETIME | 更新时间 |

---

## 智能匹配算法数据流

```
用户发布求购需求 (demand)
        │
        ▼
提取匹配特征 ──────────────────────────────────────┐
  ├── category_id  →  分类匹配 (goods.category_id)   │
  ├── keywords     →  关键词匹配 (goods.title/desc)  │
  ├── min/max_price → 价格匹配 (goods.price)        │
  ├── expected_condition → 成色匹配 (goods.condition_level) │
  └── expected_location  → 地点匹配 (goods.trade_location)  │
        │                                              │
        ▼                                              │
多特征加权融合计算匹配分数 match_score                     │
        │                                              │
        ▼                                              │
生成匹配等级 match_level + 匹配原因 match_reason          │
        │                                              │
        ▼                                              │
写入 recommend_record 表 ◄─────────────────────────────┘
        │
        ▼
前端展示匹配结果
```
