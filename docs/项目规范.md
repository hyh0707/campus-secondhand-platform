# 项目规范

## 命名规范

### 后端 Java 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 包名 | 全小写，点分隔 | `com.campus.secondhand` |
| 类名 | 大驼峰 PascalCase | `UserController`, `ProductService` |
| 方法名 | 小驼峰 camelCase | `getUserById()`, `createOrder()` |
| 变量名 | 小驼峰 camelCase | `userId`, `productName` |
| 常量 | 全大写，下划线分隔 | `MAX_UPLOAD_SIZE`, `DEFAULT_PAGE_SIZE` |
| 实体类 | 对应数据库表名，大驼峰 | `User`, `Product`, `Order` |
| DTO | 实体名 + DTO | `UserDTO`, `ProductDTO` |
| VO | 实体名 + VO | `UserVO`, `ProductVO` |
| Mapper | 实体名 + Mapper | `UserMapper`, `ProductMapper` |
| Service 接口 | 实体名 + Service | `UserService`, `ProductService` |
| Service 实现 | 实体名 + ServiceImpl | `UserServiceImpl` |
| Controller | 实体名 + Controller | `UserController`, `ProductController` |
| 工具类 | 功能名 + Util / Utils | `JwtUtil`, `FileUploadUtil` |
| 配置类 | 功能名 + Config | `CorsConfig`, `MyBatisPlusConfig` |

### 数据库命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 数据库名 | 全小写，下划线分隔 | `campus_secondhand` |
| 表名 | 全小写，下划线分隔，单数 | `user`, `product`, `order` |
| 字段名 | 全小写，下划线分隔 | `user_id`, `product_name`, `create_time` |
| 主键 | 统一 `id`，BIGINT 自增 | `id` |
| 索引 | `idx_表名_字段名` | `idx_user_phone` |
| 唯一索引 | `uk_表名_字段名` | `uk_user_phone` |

### 前端命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 组件名 | 大驼峰 PascalCase | `ProductCard.vue`, `UserLogin.vue` |
| 页面组件 | views 目录下，大驼峰 | `HomePage.vue`, `ProductDetail.vue` |
| 变量/函数 | 小驼峰 camelCase | `userInfo`, `fetchProductList()` |
| 常量 | 全大写，下划线分隔 | `API_BASE_URL`, `PAGE_SIZE` |
| CSS 类名 | 短横线 kebab-case | `.product-card`, `.user-info` |
| 路由路径 | 短横线 kebab-case | `/product-detail`, `/my-order` |
| 文件目录 | 小写，短横线分隔 | `product-detail`, `user-center` |

## 接口规范

### RESTful API 设计

| 操作 | 方法 | URL 示例 |
|------|------|----------|
| 查询列表 | GET | `/api/v1/products` |
| 查询详情 | GET | `/api/v1/products/{id}` |
| 新增 | POST | `/api/v1/products` |
| 修改 | PUT | `/api/v1/products/{id}` |
| 删除 | DELETE | `/api/v1/products/{id}` |

### 统一返回格式

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {},
  "timestamp": 1718886400000
}
```

### 状态码约定

| 状态码 | 含义 |
|--------|------|
| 200 | 成功 |
| 400 | 参数错误 |
| 401 | 未登录 / Token 过期 |
| 403 | 无权限 |
| 404 | 资源不存在 |
| 500 | 服务器错误 |

### 分页请求格式

```json
{
  "page": 1,
  "size": 10,
  "keyword": "搜索关键词",
  "categoryId": 1,
  "sort": "price_asc"
}
```

### 分页返回格式

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "total": 100,
    "page": 1,
    "size": 10,
    "records": []
  }
}
```

## 代码规范

### 后端

- 使用 Lombok 简化代码（`@Data`, `@Slf4j`, `@Builder` 等）
- Controller 层只做参数校验和路由，业务逻辑在 Service 层
- 禁止在循环中操作数据库，使用批量操作
- 日志使用 Slf4j，禁止 `System.out.println`
- 重要操作记录日志（登录、发布、下单、删除）
- 敏感信息（密码、Token）不打印到日志
- 使用 `@Validated` 进行参数校验
- 使用 `@Transactional` 管理事务

### 前端

- 组件拆分原则：单一职责，可复用
- 状态管理：全局状态用 Pinia，组件内状态用 `ref/reactive`
- API 调用统一封装在 `api/` 目录下
- 公共组件放在 `components/` 目录
- 页面级组件放在 `views/` 目录
- 路由配置统一在 `router/index.js`
- 使用 `<script setup>` 语法糖
- 使用 CSS Modules 或 `<style scoped>` 避免样式污染

### Git 提交规范

```
<type>(<scope>): <subject>

feat: 新功能
fix: 修复 Bug
docs: 文档更新
style: 代码格式（不影响功能）
refactor: 重构
test: 测试
chore: 构建/工具变动
```

## 安全规范

- 密码使用 BCrypt 加密存储
- JWT Token 设置合理过期时间（24h）
- 敏感接口需要登录校验
- 文件上传限制类型和大小
- SQL 使用 MyBatis-Plus 参数绑定，防止注入
- 跨域白名单配置
- 接口限流防护
